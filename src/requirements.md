## 요구사항 개요

- **프로젝트 이름**: ZamTok (가칭)
- **구성 요소**: `SERVER`, `Redis Service`

## SERVER 요구사항

- **채팅방 생성**
  - 채팅방은 **id**와 **pw**로 생성한다.
  - 채팅방 생성 시 다음 정보를 서버가 관리해야 한다.
    - 채팅방 id
    - 채팅방 비밀번호(pw)
    - 생성 시간
    - 생성 유저 정보(있다면)
- **채팅방 입장**
  - 다른 유저는 채팅방 **id**로 검색할 수 있다.
  - 검색된 채팅방에 대해 **pw**를 올바르게 입력하면 입장할 수 있다.
  - pw가 일치하지 않을 경우 입장에 실패하고, 적절한 에러를 반환한다.

## Redis Service 요구사항

- **채팅방 정보 저장**
  - 유저가 채팅방을 `id`, `pw`로 생성하면 Redis에 해당 정보를 저장한다.
  - 저장 정보 예시:
    - key: `room:{id}`
    - value: `pw`, 생성 시간, 기타 메타데이터
- **메시지 저장**
  - 채팅방에서 유저가 채팅을 보내면 Redis에 메시지를 기록한다.
  - 메시지는 시간 순으로 정렬 가능한 형태로 저장한다.
    - 예시: `list` 또는 `stream` 구조 사용 (추후 설계 상세화)
  - 저장 정보 예시:
    - 메시지 내용
    - 보낸 유저 id
    - 보낸 시간(timestamp)
    - 소속 채팅방 id

## 추후 상세 설계 필요 항목

- **인증/권한**
  - 유저 식별 방법(로그인 유저 vs 비로그인 손님 등)
  - 채팅방 생성/입장 권한 정책
- **Redis 키 설계**
  - 채팅방, 유저, 메시지에 대한 키 네이밍 컨벤션
  - 만료 정책(TTL) 적용 여부
- **확장성/성능**
  - 채팅방/메시지 개수가 많아질 때의 성능 전략
  - 백업 및 장애 복구 전략
